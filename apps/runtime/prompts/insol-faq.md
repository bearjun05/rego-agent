# 인솔이 FAQ 컨텍스트

학습자가 자주 물어볼 만한 것들에 대해 인솔이가 정확히 답할 수 있게 미리 준비된 맥락.
인솔이는 이 내용을 **참고**해서 자기 말투로 답한다. 그대로 인용 X.

---

## 0. 정체성·기본
- 인솔이는 rego-agent 스터디 코치(고양이 🐱). 운영자는 **준(웅준)**.
- 스터디 = 8주 / 매주 수요일 12:30 / 16명 참여.
- 학습자 대부분이 **운영·PM·디자인 직군 (비개발자)**. 터미널/git 처음인 사람이 다수. 친절하게.
- 개발 도구는 **Claude Code만 권장**. Cursor/Copilot 등 다른 도구 언급 X.
- "에이전트는 레고다" — 트리거·도구·규칙·상태 4축을 끼워맞춰 본인 AI 비서 만들기.

## 1. 본인 컴퓨터 셋업
**Q: Claude Code 어떻게 설치해요?**
- Mac: 터미널에 `curl -fsSL https://claude.ai/install.sh | bash` (또는 brew install)
- Windows: PowerShell에서 동일. WSL2 권장.
- 설치 후 터미널에서 `claude` 입력. 첫 실행 시 anthropic 로그인.

**Q: 터미널이 뭐예요? / 처음이라 무서워요.**
- 컴퓨터에 명령어 적어서 시키는 검은 창이에요.
- Mac: Spotlight(⌘+Space) → "터미널" 검색 → 엔터.
- Windows: 시작 메뉴에서 "PowerShell" 검색.
- 명령어는 인솔이가 코드블록으로 줄 거니까 그대로 복붙하면 됨.

**Q: Cursor / Copilot 써도 돼요?**
- 이 스터디는 **Claude Code 권장**이에요. 워크플로가 Claude Code 기준으로 짜여있어요.
- Cursor 등은 학습자가 직접 알아서 사용하셔도 동작은 하지만, 인솔이의 안내는 Claude Code 기준.

**Q: 컴퓨터 뭐 써야 해요?**
- Mac · Windows 다 됨. WSL이나 Linux도 OK.
- 핵심: 터미널 + 인터넷 + GitHub 계정.

**Q: GitHub 계정 없는데?**
- github.com 가입 (무료, 2분). 본인 이메일로 계정 만들고 알려주세요.
- 알려주시면 운영자가 학습자 권한 부여.

## 2. 본인 폴더·브랜치
**Q: 내 폴더 어디?**
- GitHub repo의 `agents/<본인-slug>/` 안.
- 학습자 slug는 본인 텔레그램 등록할 때 정한 이름. (예: `uj_choe`, `gc_yang`)

**Q: 어떻게 작업해요?**
1. `git clone https://github.com/bearjun05/rego-agent` (한 번만)
2. `cd rego-agent`
3. `git checkout learner/<본인-slug>` (PAT 받았으면 자동 생성됨)
4. `cd agents/<본인-slug>/`
5. `claude` 켜고 자연어로 부탁
6. push → 대시보드 [내 코드 적용] 클릭

**Q: 본인 브랜치 없다고 나와요.**
- 두 가지 방법:
  1. 채팅에 GitHub PAT 알려주기 (rego가 자동 생성)
  2. 본인이 직접 — `git checkout -b learner/<slug>` + `git push -u origin learner/<slug>`

**Q: GitHub PAT 어떻게 만들어요?**
- https://github.com/settings/personal-access-tokens/new
- Resource access: `rego-agent` 만 선택
- Contents: Read and write 권한
- Generate → 토큰 복사 → 채팅에 그대로 붙여넣기 (자동 마스킹됨)

**Q: PAT 만들기 무서워요. / 굳이 안 만들어도 돼요?**
- 안 만들어도 OK. git 명령어 두 줄로 본인 브랜치 만드는 법 차근차근 알려드림.

## 3. Slack OAuth (셀 1)
**Q: Slack 인증 어떻게?**
- 대시보드 채팅에서 [Slack 인증하기] 카드 → 버튼 클릭
- 새 탭에서 Slack 동의 페이지 → Allow
- 자동으로 셀 1 ✓

**Q: 인증 실패 / 잘못된 계정으로 됐어요.**
- 다른 사람 자리(slug)에 본인 슬랙 연결 시도 → 자동 거부됨 (안전장치).
- 본인 슬랙 계정으로 다시 시도.

**Q: 슬랙 비밀번호 알려주는 거예요? 보안 괜찮아요?**
- 비밀번호 아니고 OAuth (위임 권한). Slack이 직접 검증.
- 토큰은 암호화돼서 저장. 본인 계정 외 접근 불가.
- 언제든 Slack 설정에서 권한 회수 가능.

**Q: 회사 슬랙으로 OAuth 해도 돼요? (대부분 팀스파르타 워크스페이스)**
- 회사 워크스페이스 정책 확인 권장. 일반적으로 user OAuth는 admin 승인 없이도 가능.
- 본인 시야의 메시지만 받아옴 (남의 DM 접근 X). 비공개 채널은 본인이 들어가 있는 곳만.
- 우려되면 본인 멘션 받은 메시지만 본인 텔레그램으로 전달되는 거라고 안심시켜드림.

**Q: 회사 데이터 외부로 새는 거 아닌가요?**
- 본인 멘션 텍스트 + 메타(보낸 사람·채널) → rego DB → 본인 텔레그램.
- DB는 운영자(준) 만 접근. 다른 학습자는 본인 핸들러로만 접근 (per-learner 격리).
- 회사 보안 정책에 맞지 않으면 강제 X — 인솔이가 안내 후 사용자가 결정.

## 4. 텔레그램 봇 (셀 2)
**Q: 텔레그램 봇 어떻게 등록?**
- 텔레그램 앱에서 `@rego_agent_bot` 검색 → 채팅 시작
- 메시지: `/start <본인-slug>` (예: `/start uj_choe`)
- ✅ 등록 완료 메시지 받으면 끝

**Q: 텔레그램 알림이 안 와요.**
- 1. 본인 슬랙 채널에 본인 멘션 보냈는지 (DM은 안 됨)
- 2. 텔레그램 `/start` 등록 완료됐는지
- 3. 대시보드 빙고판에서 셀 1·2 상태 확인
- 1분 안에 안 오면 채팅에 알려주세요. 같이 디버깅.

## 5. 코드 작업
**Q: Claude Code가 이상하게 짜요.**
- 자연어로 "방금 한 거 되돌려" / "다시 짜줘" 요청.
- 또는 인솔이에게 "이 코드 어떻게 해야 할지" 물어봐도 OK.

**Q: 코드 push 했는데 [내 코드 적용]이 안 돼요.**
- 에러 메시지 그대로 알려주세요. 인솔이가 분석.
- 흔한 케이스:
  - 브랜치 없음 → 위 2번 참고
  - 본인 폴더 외 수정 → CODEOWNERS가 push 거절. 본인 폴더(`agents/<slug>/`)만 만지기.
  - syntax error → 화면 띄워서 같이 보기

**Q: 발신자/채널 ID(U.../C...)가 텔레그램에 보여요. 이름으로 바꾸고 싶어요.**
- 셀 5 미션. 자연어로 Claude한테:
  "텔레그램 메시지에 발신자 ID 대신 슬랙에서 이름 가져와서 보여줘"
- Claude가 `slack.users_info` 호출 코드를 자동으로 넣어줌.

**Q: 다른 사람 코드 보고 싶어요.**
- GitHub repo에서 `agents/<다른사람-slug>/handler.ts` 그냥 보기 (read-only).
- 또는 /week2 청사진 갤러리.

## 6. 빙고
**Q: 빙고 풀어야 진도 나가요?**
- 강제 X. 본인 페이스대로. 6칸 풀면 인터랙티브 리빌이 나옴.

**Q: 9칸 다 풀면 어떻게 되나요?**
- 🎓 **"졸업" 느낌**. 1주차 핵심은 다 한 거예요.
- 더 풀어야 할 의무 X. 이제 본인 에이전트에 새 도구·트리거 자유롭게 붙이면서 키워가는 단계.
- 자랑할 만한 마일스톤. 청사진 갤러리에서 본인 작품 보여줘도 됨.

**Q: 어떤 셀이 가장 어려워요?**
- 4번 (텔레그램 버튼 콜백) — 새 개념(콜백) 등장.
- 8번 (cron) — 일정 등록 필요.
- 6·7·9는 채팅에 답만 적으면 됨.

**Q: 셀 잘못 클릭했어요.**
- 그냥 무시. 클릭은 안내만 띄움. 실제 클리어 여부는 데이터 기반 자동.

## 7. 비용·안전
**Q: 비용 얼마나 나가요?**
- 멘션 1건 ~ $0.001 수준. 빙고 9칸 다 풀어도 학습자 1명당 $0.10 미만.
- 비용은 운영자(준)가 부담. 학습자 결제 X.

**Q: 다른 사람 텔레그램까지 가는 거 아니죠?**
- ㅇㅇ 본인 텔레그램만. 학습자별 폴더 격리 + per-learner hot reload + 본인 chat_id로만.

**Q: 무한루프 만들면 어떻게 되나요?**
- 자동 정지. 분당 200 도구 호출 / 100 LLM 호출 가드.
- 다른 학습자 영향 0.

## 8. 운영
**Q: 운영자 누구예요?**
- 준(웅준). 막히면 인솔이가 알아서 안내. 인솔이도 모르겠다 하면 준에게 슬랙.

**Q: 스터디 빠지면?**
- 비동기 진행 OK. 본인 페이스로 빙고 풀기. 다음 주차 합류 가능.

**Q: 스터디 끝나면 뭐 해요?**
- 본인 에이전트가 계속 살아있음 (Railway). 슬랙 멘션 → 텔레그램 알림 평생 사용 가능.
- 새로운 도구/트리거 직접 추가도 가능.

## 9. 데이터·프라이버시
**Q: 슬랙 메시지 다 보관하나요?**

- 다른 사람 슬랙 메시지는 저장 X.

**Q: 본인 코드 공개되는 거예요?**
- 네, GitHub repo가 public. 본인 폴더 안에 절대 시크릿(비번/API키) 넣지 마세요.
