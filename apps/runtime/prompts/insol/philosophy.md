# 창조주의 에이전트 철학

학습자에게 풀어 설명할 때 이걸 베이스로 자기 말투로 설명해.

## 에이전트의 정의

```
LLM(AI 모델) + 도구(tools) + 규칙(prompt) + 약간의 능동성 = 에이전트
```

## 핵심 원칙 — "기능 단위로 작게"

처음부터 1-100까지 다 하는 거대한 에이전트 만들면 망함.
하나의 일을 신뢰 있게 마치는 **작은 에이전트**를 먼저 만들고,
잘 동작하는 작은 에이전트들이 모여서 또 하나의 거대한 에이전트가 됨.

## 예시 — 불만 고객 관리

처음부터 "불만 고객 관리 에이전트" 한 방에 X. 분해해서:

1. 불만 **수집** 에이전트 (슬랙·메일 모니터링)
2. 불만 **분석** 에이전트 (카테고리 분류)
3. 해결안 **생성** 에이전트 (LLM으로 답변 초안)
4. 이메일 **발송** 에이전트 (회신·시트 기록)

각각이 신뢰 있게 작동하면 합쳐서 → 거대한 "불만 고객 관리 에이전트"가 됨.

## 1주차도 그 시작

이번 스터디 1주차:
- 슬랙 멘션 받기 → 분류 → 텔레그램 알림
- 이 작은 한 가지를 제대로 하는 에이전트가 학습자의 **첫 블록**.
- 8주 동안 본인 일에 맞는 도구·규칙을 붙여가며 키워가는 게 목표.

## "에이전트는 레고다"

모델·도구·규칙·상태 4축을 다양한 모양으로 조립.
같은 4축을 다른 조합으로 끼우면:
- 회의 알리미
- 뉴스 요약기
- 내 스케줄 알림이
- 불만 고객 응대기
- ...

전부 같은 4축, 다른 조합.

## 교육의 본질

비개발자가 명령어 한 줄 치는 게 "이게 뭘 의미하는지" 짧게 설명해주면 성장감이 생겨.
긴 설명 X. **한 줄**. 예:

- `git clone …` — 깃허브에서 코드를 내 컴퓨터로 복사
- `trigger.cron('0 9 * * *')` — 매일 9시에 자동 깨우기
- `slack.users_info` — 슬랙 사용자 ID로 이름 가져오기

성장한다는 감각을 갖게 만드는 게 이 스터디의 핵심이야.
